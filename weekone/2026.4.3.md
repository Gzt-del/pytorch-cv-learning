## 一、今日学习知识点
- **PyTorch 核心概念**  
  PyTorch 是开源深度学习框架，支持 GPU 加速，是机器学习、深度学习开发的核心工具，本次安装为后续模型搭建打下环境基础。

- **环境依赖与版本匹配**  
  前置环境：必须先安装 Python（推荐 3.8~3.11 版本）、Conda/venv 虚拟环境（隔离项目依赖，避免版本冲突）；  
  核心关联：PyTorch 版本需要与 CUDA 版本（GPU 加速）、操作系统（Windows/Linux/Mac）严格匹配；  
  相关库：安装时会自动配套 torchvision（计算机视觉工具）、torchaudio（音频处理）、numpy（数值计算）等依赖库。

- **两种主流安装方式**  
  Conda 安装：适合 Anaconda/miniconda 用户，自动解决依赖冲突，稳定性更高；  
  Pip 安装：适合纯 Python 环境，命令简洁，适配性广；  
  官方安装命令生成：必须通过 PyTorch 官网根据自身系统、CUDA 版本生成专属命令，不可随意复制。

- **环境验证方法**  
  学会通过 Python 终端执行代码，验证 PyTorch 是否安装成功、是否启用 GPU 支持。

## 二、遇到的问题
- **虚拟环境创建失败**  
  初次使用 conda 创建虚拟环境时，命令输入错误，提示 `command not found`。

- **PyTorch 安装速度极慢 / 下载失败**  
  直接使用国外官方源安装，网络超时，安装包下载中断。

- **CUDA 版本不匹配**  
  电脑显卡支持 CUDA 11.8，但错误复制了 CUDA 12.1 的安装命令，导致安装后无法调用 GPU。

- **安装后导入 torch 报错**  
  提示 `ModuleNotFoundError: No module named 'torch'`，明明执行了安装命令却找不到库。

## 三、解决方法
- **虚拟环境问题解决**  
  原因：未正确配置 conda 环境变量，或命令拼写错误；  
  方法：重新打开 Anaconda Prompt（Windows），执行标准创建命令。

- **安装慢 / 失败问题解决**  
  切换国内清华镜像源，加速下载。
